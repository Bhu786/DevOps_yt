When reading a domain name from right to left, focus on the main domain and the `.com` extension. For example, in `www.example.com`, `.com` is the extension and `example` is the main domain. Subdomains and other parts can usually be ignored.
Ex : XXXX.bank.in ==> bank.in main top extension


When you access a website like `google.com`, your browser converts the domain name into an IP address. The server receives the IP address, not the domain name. This process is called DNS resolution.


For example, the IP address for `google.com` is often `142.250.190.78`. You can access Google by entering this IP address directly in your browser's address bar: `http://142.250.190.78`. Note that some websites may not work properly when accessed by IP address due to security or configuration settings.

### Difference Between Cloud and On-Premise

Cloud and on-premise solutions both store and manage data, but the main difference is where the infrastructure is located and how it is accessed.

- **On-Premise:** The servers and data are hosted locally within your organization's premises. Access is usually restricted to internal users, and you control the hardware and network.
- **Cloud:** The servers and data are hosted by a cloud provider and accessed over the internet. Public cloud services provide access to anyone with the proper credentials, and the infrastructure is managed by the provider.

In cloud setups, resources are available via public IP addresses and can be accessed from anywhere, while on-premise setups are typically limited to local access unless special networking is configured.
